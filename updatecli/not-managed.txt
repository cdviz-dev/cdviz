# Dependencies NOT managed by updatecli
#
# This file documents dependencies that are intentionally excluded from
# automated updates. Each entry explains why it cannot or should not be
# auto-updated.

## Docker images — major-version pinned
# These images are pinned to a major version because upgrading across major
# versions requires manual testing and/or configuration changes.

# timescale/timescaledb-ha:pg18  (demos/stack-compose/docker-compose.yaml)
#   Pinned to pg18 (PostgreSQL 18). A major PostgreSQL upgrade requires schema
#   compatibility review and a manual migration runbook.

# migrate/migrate:4  (cdviz-db/Dockerfile)
#   Pinned to major version 4. The CLI interface may change between majors.

## Docker images — internal / always-latest
# Internal images are always pulled as :latest and are released independently.

# ghcr.io/cdviz-dev/*:latest  (demos/stack-compose/docker-compose.yaml)
#   Internal project images; always :latest in compose to pick up the most
#   recent build without a version bump in this repo.

## Generated manifests
# These files are auto-generated by external tooling and must not be edited
# directly.

# demos/stack-k8s/  (all generated Helmwave manifests)
#   Rendered by `helmwave build`. Editing the YAML directly would be
#   overwritten on the next build.

## Mise tools — unmanageable by updatecli
# helmwave (demos/stack-k8s/mise.toml)
#   Configured in mise as an inline TOML table under the key
#   [tools."ubi:helmwave/helmwave"] with special characters (colon, slash).
#   Neither a file regex (pattern mismatch) nor a TOML key path (special chars)
#   can reliably target this entry. Update manually by editing the `version`
#   field in that section.

## Mise tools pinned to "latest"
# These tools intentionally resolve to the latest available version at
# install time and are therefore not version-pinned in mise.toml files.

# bun          — fast-moving runtime, always want the latest stable
# yq           — YAML processor, no breaking changes expected
# git-cliff    — changelog generator
# dprint       — code formatter
# biome        — linter/formatter
# golang-migrate — DB migration CLI (version pinned in cdviz-db/Dockerfile instead)

# updatecli    — updatecli itself (updatecli/mise.toml)
# ubi:chris48s/v8r — JSON schema validator (updatecli/mise.toml)
